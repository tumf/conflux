## 背景
- Git コマンドの実装が 1800 行超まで増え、責務の境界が曖昧になっている。
- ワークツリー/マージ/コミット関連の変更が同一ファイルに混在している。

## 目標
- Git コマンド群を責務別に分割し、保守性と変更容易性を高める。
- 既存 API の呼び出し元に影響を与えない。

## 非目標
- Git コマンドの機能追加や挙動変更。
- VCS 抽象化レイヤーの再設計。

## 方針
- commands/mod.rs を入口にし、basic/commit/worktree/merge に分割する。
- 既存の公開関数は mod.rs から再公開し、呼び出し元の変更を最小限にする。
- テストは責務別ファイルに移動し、既存挙動を担保する。

## トレードオフ
- ファイル分割により import が増える。
- 移動コストは一時的に増えるが、中長期で保守性が改善する。

## 失敗時の扱い
- テスト差分が発生した場合は分割単位を縮小し、挙動の差分を解消してから再分割する。
