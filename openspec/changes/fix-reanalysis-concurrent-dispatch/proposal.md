# Change: 再分析ループを apply 実行と並行させる

## なぜ
- apply 実行中にキューへ変更を追加しても、re-analysis が走らず queued から apply へ遷移しない。
- 現行実装ではディスパッチ処理を await するため、再分析ループが apply 完了まで停止してしまう。
- 仕様の「実行中でも監視し、空きスロットで即時に起動する」要件に反している。

## 何を変えるか
- re-analysis ループを apply 実行と並行して継続させる。
- ディスパッチは非同期タスク化し、in-flight 状態を追跡して空きスロットを即時に使えるようにする。
- queued の変更が追加された場合、apply 中でも再分析が起動し、空きスロットが出次第ディスパッチされるようにする。

## 影響範囲
- 並列実行の re-analysis/dispatch ループ
- in-flight 管理とスロット空き検知
