# Change: 追跡中バックアップファイルを整理する

## Why
`src/` 配下に `.bak` / `.backup` / `.bak2` が追跡されており、実コードと旧コピーが混在しています。検索結果やレビュー対象がノイズ化し、誤編集・誤参照リスクが高い状態です。

## What Changes
- 追跡されているバックアップ拡張子ファイルを整理し、正規ソースのみを維持する
- 同種ファイルの再混入を防ぐため、検知ルール（CI/テスト/チェック）を追加する
- 削除前に現行挙動を固定するキャラクタリゼーションテストを整備し、機能回帰がないことを確認する

## Impact
- Affected specs: `code-maintenance`
- Affected code: `src/**/*.bak*`, チェック用スクリプト/テスト
- API/CLI互換性: 変更なし

## Acceptance Criteria
- 追跡対象に `.bak` / `.backup` / `.bak2` ファイルが存在しない
- `cargo test` が成功し、既存機能に回帰がない
- 以後、同拡張子の追跡混入を自動検知できる
