# Change: サーバーモード Web UI のプロジェクト未選択ログ表示と選択解除トグル

## Problem/Context
- 現在の server-mode Web UI は、右ペインの Logs タブで常に `selectedProjectId` に紐づくログだけを表示し、未選択時は案内文のみを表示する。
- そのため、オーケストレーション全体の進行を俯瞰したいときに、プロジェクト未選択状態ではログを見られない。
- また、プロジェクトカードは再クリックしても選択解除できず、全体ビューへ戻る操作がない。

## Proposed Solution
- プロジェクト未選択時の Logs 表示を「選択してください」ではなく、全プロジェクト横断のオーケストレーション全体ログへ切り替える。
- プロジェクト選択時は従来どおり、そのプロジェクトに紐づくログだけを表示する。
- プロジェクトカードのクリックおよび Enter / Space によるアクティベーションをトグル化し、選択済みカードを再度操作した場合は選択を解除する。
- 選択解除時は、プロジェクト依存の詳細表示が未選択状態へ戻るよう、関連する UI 状態も整合させる。

## Acceptance Criteria
- プロジェクト未選択時、デスクトップ右ペインの Logs タブに全体ログが表示される。
- プロジェクト選択時、Logs タブはそのプロジェクトのログのみを表示する。
- 選択済みプロジェクトカードを再クリックすると選択が解除される。
- Enter / Space によるカード操作でも同じトグル挙動になる。
- モバイルの Logs タブでも未選択時は全体ログ、選択時はプロジェクトログになる。

## Out of Scope
- Files タブの未選択時挙動変更
- ログ永続化方式やサーバー側ログ配信プロトコルの変更
