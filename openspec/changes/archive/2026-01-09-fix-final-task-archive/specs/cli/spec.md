# cli Specification Delta

## MODIFIED Scenarios

### Parent: 実行モードダッシュボード

#### MODIFIED Scenario: 処理完了時の表示

- **WHEN** 全てのキュー内変更の処理が完了する
- **THEN** ヘッダーのステータスが「Completed」に変更される
- **AND** TUIは表示を維持し、ユーザーが `q` キーで終了できる
- **AND** 変更の完了状態をリトライ付きで確認してからアーカイブ処理を実行する

#### ADDED Scenario: apply完了後のアーカイブ確認

- **WHEN** applyコマンドが正常終了する
- **THEN** 変更の完了状態をリトライ付きで確認する
- **AND** 完了が検出された場合はアーカイブコマンドを実行する
- **AND** 最大3回のリトライ後も完了が検出されない場合は警告ログを出力する

#### ADDED Scenario: 完了状態確認のリトライ

- **WHEN** 変更の完了状態を確認する
- **AND** 完了状態でない場合
- **THEN** 500ミリ秒待機後に再確認する
- **AND** リトライ回数をログに出力する

#### ADDED Scenario: 単一変更の処理完了

- **WHEN** 単一の変更を処理する
- **AND** applyコマンドが全タスクを完了させる
- **THEN** 変更は確実にアーカイブされる
- **AND** 「Archived」状態としてTUIに表示される

#### ADDED Scenario: 複数変更の最後の処理完了

- **WHEN** 複数の変更を処理する
- **AND** 最後の変更のapplyコマンドが完了する
- **THEN** 最後の変更も確実にアーカイブされる
- **AND** 全ての変更が「Archived」状態になった後にAllCompletedイベントが送信される

## ADDED Requirements

### Requirement: 完了検出のリトライ設定

完了状態の検出においてリトライ動作を実装しなければならない（SHALL）。

#### Scenario: デフォルトのリトライ設定

- **WHEN** 設定ファイルにリトライ設定がない
- **THEN** 最大リトライ回数は3回である
- **AND** リトライ間隔は500ミリ秒である

#### Scenario: キャンセル時のリトライ中断

- **WHEN** リトライループ実行中である
- **AND** キャンセルトークンがキャンセルされる
- **THEN** リトライループは即座に終了する
- **AND** プロセスは適切にクリーンアップされる
