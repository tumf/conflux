## 背景
- Agent の実装が 1500 行超まで増え、責務の境界が曖昧になっている。
- 出力収集・履歴記録・実行ロジックが同一ファイルに混在している。

## 目標
- runner/output/history/prompt に分割し、責務と変更範囲を明確にする。
- 既存 API の呼び出し元に影響を与えない。

## 非目標
- Agent コマンドの挙動変更。
- 履歴フォーマットや出力仕様の変更。

## 方針
- agent/mod.rs を入口とし、既存関数を再公開して呼び出し元を守る。
- 分割後も挙動差分が出ないように既存テストを維持する。

## トレードオフ
- ファイル分割により import とモジュール宣言が増える。
- 移動コストは一時的に増えるが、中長期で保守性が改善する。

## 失敗時の扱い
- テスト差分が発生した場合は分割単位を縮小し、挙動の差分を解消してから再分割する。
