## 背景
- parallel モジュールが 5000 行超まで肥大化しており、責務境界と変更影響が追跡しづらい。
- テストが同一ファイルに混在しているため、レビュー時にコードとテストが混線する。

## 目標
- 並列実行の責務を workspace/queue/merge/state に分割し、可読性と保守性を改善する。
- テストをサブモジュールへ移動し、変更範囲の検証を容易にする。

## 非目標
- 並列実行の仕様や挙動の変更。
- 依存解決アルゴリズムの変更。

## 方針
- 既存 public API は mod.rs で再公開し、外部呼び出しの影響を最小化する。
- 共有データ構造は types.rs に残し、分割後も単一参照点を維持する。
- テストは executor / merge / queue など責務別にファイル分割する。

## トレードオフ
- ファイル分割による import 増加と、初回の移動コストが発生する。
- 可読性とモジュール境界の明確化により、中長期的な変更コストを下げる。

## 失敗時の扱い
- 既存テストで差分が発生した場合は、挙動が変わらないように移動を戻し、移動順序を見直す。
