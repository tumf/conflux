# Change: TUIダッシュボードの追加

## Why

現在の `run` コマンドは `indicatif` による単純なプログレスバー表示のみで、複数の変更の全体像を把握しにくい。また、処理する変更を事前に選択する機能がなく、実行中に新しい変更が追加された場合の対応もできない。インタラクティブなTUIダッシュボードを導入することで、変更の選択、進捗監視、動的なキュー管理を一画面で行えるようにする。

## What Changes

- `ratatui` + `crossterm` を使用したインタラクティブTUIを新規実装
- **デフォルト起動動作**: サブコマンドなしで起動した場合、TUIを表示
- **選択モード**: 
  - 変更一覧をチェックボックス付きで表示
  - 矢印キーでカーソル移動、Spaceで選択切り替え
  - F5で選択した変更の処理を開始
- **実行モード**: 
  - 選択した変更の進捗をダッシュボード表示
  - ログのリアルタイム表示
  - 実行中も変更の選択操作が可能（動的キュー追加）
- **自動更新機能**:
  - 5秒間隔で変更一覧を自動リフレッシュ
  - 新規変更の検出と「NEW」バッジ表示
  - 新規変更はデフォルトで未選択状態
- **動的実行キュー**:
  - 実行中に未選択の変更を選択するとキューに追加
  - 次の処理サイクルで実行される
- 現在の `indicatif` ベースの `progress.rs` を `tui.rs` モジュールに置き換え

## Impact

- Affected specs: `cli`
- Affected code:
  - `src/tui.rs` (新規)
  - `src/cli.rs` (デフォルト動作の変更)
  - `src/progress.rs` (削除または非推奨化)
  - `src/orchestrator.rs` (TUI連携、動的キュー管理)
  - `src/openspec.rs` (変更一覧の差分検出)
  - `src/main.rs` (TUI初期化、モード切替)
  - `Cargo.toml` (依存関係追加)
