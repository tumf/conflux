## Context
並列実行の制御フローがバッチ/グループ完了を前提としており、実行中のキュー追加や再分析がバッチ境界に縛られている。既存仕様はキュー変化やタイマーによる re-analysis を要求しているため、バッチ/グループ前提は構造的に矛盾している。

## Goals / Non-Goals
- Goals:
  - バッチ/グループ前提を完全に廃止し、空きスロット駆動の連続ディスパッチへ統一する
  - 依存失敗チェックやディスパッチをグループ単位ではなく個別に評価する
- Non-Goals:
  - 依存関係分析アルゴリズムや order 生成の変更
  - merge 手順や worktree 作成方式の再設計

## Decisions
- Decision: 実行ループは「空きスロット検知 → re-analysis → ディスパッチ」を継続的に評価する方式に統一する
- Decision: バッチ/グループ単位の完了イベントや完了判定は廃止し、スロット駆動の完了判定へ集約する
- Alternatives considered: バッチ境界を維持しつつデバウンスのみ短縮する（バッチ待ちが残るため不採用）

## Risks / Trade-offs
- 空きスロット検知の頻度が高い場合、re-analysis 呼び出しが増える
- デバウンスとキャッシュで過剰な再分析を抑制する必要がある

## Migration Plan
1. バッチ/グループ前提の関数と呼び出しを洗い出す
2. 連続ディスパッチへ置き換える
3. 依存失敗チェックを個別ディスパッチへ移動する
4. ログ/イベント/メッセージの語彙を更新する
