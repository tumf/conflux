## Context
AIコマンド実行はストリーミング出力を前提としており、無出力が長時間続くとユーザー視点では停止したように見える。現状はプロセス終了まで待ち続けるため、無出力監視を設けて安全に中断できるようにする。

## Goals / Non-Goals
- Goals:
  - 無出力状態を一定時間で検知し、実行を中断できる
  - 既存のリトライ判定・スタッガー動作と互換を保つ
  - 設定で閾値と無効化を制御できる
- Non-Goals:
  - LLMの内部状態可視化やハートビート生成の強制
  - 外部監視サービスとの連携

## Decisions
- Decision: CommandQueue の streaming 実行に「無出力監視」を組み込み、stdout/stderr のいずれかが到着した時点でタイマーをリセットする。
- Decision: タイムアウト時は警告ログを出し、猶予時間後に強制終了する。
- Decision: 既存の retry 判定は維持し、無出力タイムアウトは失敗として扱って同じ経路に乗せる。

## Risks / Trade-offs
- 無出力が正常な長時間処理（例: 大規模テスト）でもタイムアウトする可能性がある → 設定で無効化/延長可能にする。
- 強制終了時のプロセス後始末が必要 → 既存の ManagedChild 終了手順に合わせる。

## Migration Plan
1. 新しい設定キーを追加し、デフォルト値を設定する
2. streaming 実行に無出力監視を追加する
3. タイムアウト時のログとエラーメッセージを追加する
4. テストで無出力タイムアウトと無効化動作を確認する
