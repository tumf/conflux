# Change: 実行開始時に存在しなかったchangeを無視する

## Why

現在の実装では、オーケストレーションループの各イテレーションで `openspec list` を呼び出してchange一覧を取得している。
そのため、実行中にユーザーが新しいproposalを追加した場合、その中途半端な状態のchangeが処理対象になる可能性がある。

これにより以下の問題が発生する：
- まだ準備が完了していないchangeが実装される
- ユーザーが意図しないタイミングで変更が適用される
- proposal作成中にオーケストレーターが介入し、不完全な実装が行われる

## What Changes

- `run`コマンド開始時にchange一覧のスナップショットを取得
- 以降のイテレーションでは、スナップショット内のchangeのみを処理対象とする
- 各イテレーションでの`openspec list`呼び出しはタスク進捗の更新確認のために維持
- スナップショットに存在しないchangeは無視する（ログで通知）

## Impact

- Affected specs: なし（新規機能、破壊的変更なし）
- Affected code: `src/orchestrator.rs`
- 動作変更：実行開始後に追加されたchangeは次回の`run`まで処理されない
