## Context
TUI の停止後に parallel モードへ切り替えて再開すると、直前の停止状態が残り即時停止してしまう。停止/キャンセルの状態が実行単位でリセットされず、次回の実行開始に影響している。

## Goals / Non-Goals
- Goals:
  - 新規実行開始時に停止/キャンセル状態を必ずリセットする
  - Stopped モードでの serial/parallel 切替後でも F5 で正常に開始する
  - 停止理由が次回実行の完了判定に影響しないようにする
- Non-Goals:
  - 停止/キャンセルの操作方法やキー割り当て変更
  - 実行中の停止ロジックの動作変更（停止自体の仕様は維持）

## Decisions
- Decision: 実行開始時に停止/キャンセル状態をクリアする共通処理を追加する
  - Why: 直前の実行で設定された停止フラグが再開を阻害しているため
- Decision: モード切替後の再開でも同じリセット処理を適用する
  - Why: serial/parallel の切替が停止状態の例外にならないようにするため

## Risks / Trade-offs
- リセットタイミングを誤ると、停止直後の状態が失われる可能性がある
  - Mitigation: 「新規実行開始時」に限定し、停止中・停止完了時の状態は保持する

## Migration Plan
- 状態リセット処理を追加して既存フローに差し込む
- 既存の停止・再開テストに新しいシナリオを追加する
