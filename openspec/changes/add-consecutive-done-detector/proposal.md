# 提案: 連続完了シグナル検出（Consecutive Done Signal Detector）

## 概要

Ralph-claude-codeの終了検出アルゴリズムを参考に、エージェントが2回連続で「完了」を示すレスポンスを返した場合、そのchangeを完了と判定する機能を追加します。

## 背景

Ralph-claude-codeでは、AIエージェントが2回連続で「完了」シグナル（doneメッセージ）を返した場合、タスクが完了したと判定します。これにより、タスクMDのチェックマークが更新されていなくても、エージェントの判断で完了を認識できます。

現在のOrchestratorは、tasks.mdのcompleted_tasks/total_tasksのみで完了を判定しており、エージェント自身の判断を考慮していません。

## 目的

- エージェントの完了判断を尊重
- tasks.md更新なしでも完了検出可能
- より柔軟な完了判定メカニズム

## 影響範囲

- `src/agent.rs`: エージェント出力から完了シグナルをパース
- `src/orchestrator.rs`: 連続完了シグナルのカウントと判定
- `src/config/mod.rs`: 完了シグナルしきい値設定

## リスク

- 中リスク: エージェント出力フォーマット依存
- 誤検知リスク: エージェントが誤って完了と判断する可能性

## 代替案

- tasks.mdベースの判定のみ（現状維持）
- 完了シグナル + tasks.md両方の条件を満たす場合のみ完了とする
