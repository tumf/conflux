## 背景
- events.rs が 2000 行超まで増え、イベント処理の責務が混在している。
- AppState のイベント更新ロジックが集中しており、変更時の影響範囲が広い。

## 目標
- イベント種別ごとにモジュールを分割し、責務境界を明確化する。
- AppState の公開 API は維持し、呼び出し側の変更を最小限にする。

## 非目標
- TUI の表示仕様やステータス遷移の変更。
- AppState 構造体の再設計。

## 方針
- events/mod.rs を入口にし、processing/completion/progress/refresh などのサブモジュールに分割する。
- 分割後も handle_orchestrator_event のシグネチャは維持する。
- テストはイベント種別ごとに整理し、必要な追加テストで同一性を保証する。

## トレードオフ
- 分割直後は import とモジュール宣言が増える。
- 可読性向上と責務分離により、将来の変更コストを下げる。

## 失敗時の扱い
- イベント処理に差分が出た場合、分割対象を小さく戻し、挙動差分を解消してから再分割する。
