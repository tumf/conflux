# Change: Webダッシュボードのステータス自動更新を修正する

## Why
HTTP サーバー（Webダッシュボード）のステータス表示が更新されず、手動リロードしても最新状態が反映されません。監視用途として信頼できる表示にするため、自動更新と“古い状態の表示”を確実に避ける挙動を明確化します。

## What Changes
- Webダッシュボードがオーケストレーター状態を自動で更新する（WebSocket を主経路、フォールバックでポーリング）
- `GET /api/state` を含む状態取得が、ブラウザキャッシュ等で stale にならないことを保証する（HTTP キャッシュ抑止）
- 初期ロード時に最新スナップショットを必ず描画し、更新が発生しない“静止状態”でも表示が取り残されないようにする

## Impact
- Affected specs: `web-monitoring`
- Affected code (想定): HTTP サーバーの状態取得/API レスポンスヘッダ、WebSocket ブロードキャスト、ダッシュボードの JavaScript 更新ロジック
- Compatibility: 既存 UI/CLI の破壊的変更なし（表示と更新の信頼性改善）

## Out of Scope
- ダッシュボードの UI デザイン刷新
- 新しい API エンドポイントの追加（必要になった場合は別提案）
