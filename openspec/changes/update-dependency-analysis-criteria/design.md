## Context
依存関係分析は LLM によるプロンプト解析に依存しており、現状では order の推奨順位と dependency の必須条件が曖昧なまま返されることがあります。これにより、本来並列実行できる change が依存関係として扱われ、実行が滞留する問題が発生しています。

## Goals / Non-Goals
- Goals:
  - dependency を「成立に必須な関係」に限定する
  - order は推奨実行順序として独立に返させる
- Non-Goals:
  - 並列実行のアルゴリズム自体の変更
  - 依存関係推論の自動検証ロジック追加

## Decisions
- Decision: 依存関係の定義をプロンプトに明記し、order と依存関係の区別を明文化する
- Alternatives considered: 依存関係出力の自動補正ロジック導入
  - 理由: 出力補正は誤判定の根本原因を隠し、依存関係の意味がさらに不透明になるため見送る

## Risks / Trade-offs
- 依存関係が過少に出力されると実行順序が不安定になる可能性がある
- ただし order に推奨順位を残すことで、実行順の安定性は確保できる

## Migration Plan
1. 依存関係分析プロンプトの更新
2. 既存の依存関係判断に影響がないことを確認
