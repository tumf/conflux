# Change: 依存関係判定の厳格化とorder分離

## Why
依存関係分析が「優先度や並び順の好み」を依存関係として扱うケースがあり、並列実行の待ち状態が過剰に発生しています。依存関係と実行順序（order）は別概念であるため、依存関係は必須条件に限って厳密に導出し、orderは推奨順序として扱う必要があります。

## What Changes
- 依存関係の定義を「片方の change が他方の成果物・仕様・APIに明確に依存し、それがないと成立しない場合」に限定する
- order は優先度・効率・進行状況などを踏まえた推奨実行順序として扱い、依存関係とは独立に記述する
- 依存関係分析プロンプトの指示を更新し、依存関係と order の混同を防ぐ

## Impact
- Affected specs: parallel-analysis
- Affected code: 依存関係分析プロンプト生成、並列実行分析のガイドライン
