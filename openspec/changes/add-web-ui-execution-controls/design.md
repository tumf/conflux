## Context
Web監視UIにTUI相当の実行制御を追加し、TUIとRunの両方で同一の制御経路を利用する。WebSocketで配信される`app_mode`を拡張し、UIが停止/停止キャンセル/強制停止を正しく表現できるようにする。

## Decisions
- Web制御APIはHTTP POSTの専用エンドポイントで提供する
- 既存のTUI実行制御ロジックと同じ経路を再利用し、二重実装を避ける
- app_mode語彙はTUIと一致させ、Web UIも同じ判定を使用する

## Risks / Trade-offs
- Web APIの誤操作が実行に影響するため、ログとエラーレスポンスで失敗を明示する
- 実行開始対象の選択状態は既存のTUI選択状態に依存する

## Security
- 追加の認可やトークンは導入せず、既存の `--web-bind` の警告とバインド制御に従う
- 外部公開する場合は `--web-bind 127.0.0.1` を維持する（仕様外の運用）
