# Change: Reanalysis Trigger Redesign

## なぜ
- 動的キュー追加後の再分析が「前のグループ完了」に依存しており、完了イベントが発生しない限り再分析が開始されない。
- その結果、10秒経過しても re-analysis が始まらないなど、期待される応答性が得られない。

## 何を変えるか
- 再分析の起動条件を「グループ完了」から切り離し、キュー変化やタイマーを起点とする連続スケジューラに改める。
- 動的キューをイベント駆動で通知できるようにし、再分析ループが即時反応できるようにする。
- スロットが空いていない場合は再分析結果を保持し、空きができ次第ディスパッチする方式に変更する。

## 影響範囲
- 並列実行の再分析トリガとディスパッチの制御ロジック
- TUI 動的キューの通知/監視方法
