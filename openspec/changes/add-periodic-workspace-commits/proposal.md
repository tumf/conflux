# Change: 定期的なワークスペースコミット

## Why

並列実行モードでは、各changeに対してworkspaceを作成してapplyループを実行する。現在の実装では、applyループが完了（または最大イテレーション到達）するまでコミットが作成されない。

この設計には以下の問題がある：

1. **進捗の喪失**: 最大イテレーション（50回）に達した場合、途中まで進んでいたタスクの作業結果がコミットされていないため、workspaceクリーンアップ時に失われる
2. **復旧の困難**: `add-workspace-resume` 機能と組み合わせても、コミットされていない変更は復旧できない
3. **可視性の欠如**: 途中経過がVCS履歴に残らないため、問題発生時のデバッグが困難

## What Changes

- 各applyイテレーション終了時に自動的にコミット/スナップショットを作成
- jj: `jj describe` で進捗をコミットメッセージに反映
- Git: `git commit` で定期的にコミット
- コミット頻度を設定可能に（デフォルト: 毎イテレーション）

### 詳細

1. **イテレーション終了時のコミット**: applyコマンド実行後、タスク進捗をチェックした後にコミットを作成
2. **コミットメッセージ**: `WIP: {change_id} ({completed}/{total} tasks)` 形式
3. **設定オプション**: `commit_interval` で N イテレーションごとにコミットするか指定可能

## Impact

- 影響するspec: `parallel-execution`
- 影響するコード:
  - `src/parallel/executor.rs` (applyループ内にコミット処理追加)
  - `src/config/mod.rs` (commit_interval設定追加)
